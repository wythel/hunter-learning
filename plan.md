# Hunter Learning — 兒童學習遊戲網站 計畫

## 專案目標

打造一個以手機為主的靜態網站，包含多個互動小遊戲與視覺化學習頁面，幫助孩子透過遊戲方式吸收知識。

---

## 技術選型

- **純 HTML / CSS / JavaScript**（無框架，方便在任何地方直接開啟）
- 或 **Vite + React**（若遊戲互動邏輯較複雜時採用）
- 部署：GitHub Pages 或 Netlify（免費、簡單）
- 不需後端，全部靜態

---

## 網站架構

```
index.html          ← 首頁：所有遊戲的入口卡片列表
/games/
  game-01/          ← 每個遊戲各自一個資料夾
    index.html
    style.css
    script.js
  game-02/
    ...
/shared/
  style.css         ← 共用樣式（手機優先）
  nav.js            ← 返回首頁等共用元件
```

---

## 手機優先設計原則

- Viewport meta tag 設定
- 字體大小 >= 16px，按鈕點擊區域 >= 44px
- 不依賴 hover，改用 tap/touch 互動
- 畫面以垂直捲動為主，避免橫向捲動
- 配色鮮明、對比度高，適合兒童視覺

---

## 首頁設計

- 大卡片式網格，每張卡片代表一個遊戲
- 卡片顯示：遊戲名稱、簡短說明、難度/年齡建議
- 點擊卡片進入對應遊戲頁面

## 開發流程

1. 建立首頁框架與共用樣式
2. 每次新增一個遊戲：先確認規則與視覺，再實作
3. 在手機上測試，確認觸控操作順暢
4. 累積到一定數量後考慮加入分類篩選功能
